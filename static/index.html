        Welcome to PiScope Radar

        This dashboard pulls ADS-B traffic from your PiAware (or adsb.lol) and renders it as a live radar map, sidebar, and per-aircraft detail panel.

        A quick 30-second tour to point it at your location:

        • Step 1: Tell PiScope where your receiver is. The distance ring, polar coverage, and the global-feed centre all depend on this.
        • Step 2: Pick a contact URL for outbound HTTP requests. Some upstream APIs reject anonymous traffic.
        • Step 3: Done — you can change anything later in Settings.

        Where's your receiver? Either type the coordinates or click on the map.

        Some upstream APIs (notably planespotters.net) now require a contact URL or email in the User-Agent header. PiScope uses this one for every outbound request.

        Default points at this repo — fine for a personal install. Change it to your own URL once you're up and running.

        ✅ You're set up.

        • Press ? any time for the keyboard shortcut list.
        • Right-click anywhere on the map to set receiver coords, copy lat/lon, or follow an aircraft.
        • Share a URL like /piscope#center=55,-3&zoom=8&select=HEX — the recipient lands on the exact same view.

            Feed mode

            Or right-click your spot on the map. Both update the receiver and the global-feed centre at once.

            Extra feeds (multi-receiver)

            Each additional tar1090 base URL will be polled in parallel and merged into the live view by aircraft hex. Format: Name|http://host/tar1090, one per line.

            Most options are free with no key required. Tactical / radar themes look best on dark or satellite bases.
            Full mode auto-hides labels below the zoom threshold so busy airspace stays readable.

            Aviation overlay (OpenAIP)

            Free for personal use — sign up at openaip.net, create a "Client API key" in your account dashboard, paste it here. Tiles are proxied through the Pi so the key never leaves your server.

            AI explanations

            Pick a backend for the "Explain this aircraft" button and the daily digest commentary. Three options: a local LLM via Ollama, a cloud API (your own Anthropic / OpenAI / Google key), or a Claude CLI shim that uses an existing Claude Code subscription. The Explain button hides itself when the selected provider isn't configured.

            Each provider keeps its own config below — switching providers doesn't wipe the others' settings.
            Most-recent N exchanges are stitched into each follow-up prompt. Higher = more context; ~doubles per-message tokens. Free for Ollama, billed for Cloud API / Claude CLI.
            Must be reachable from the Pi. Typical setup: OLLAMA_HOST=0.0.0.0:11434 ollama serve on the host.
            Model tag as ollama list reports it.
            Empty falls back to a cheap-and-capable default per vendor: claude-haiku-4-5, gpt-4o-mini, or gemini-2.5-flash.
            Points at a claude-shim daemon (see tools/claude-shim/ in the repo).

            Webhooks

            Forward events to Discord, Slack, ntfy.sh, or any generic JSON endpoint. POSTs are fire-and-forget — a failed delivery never stalls the feed loop.

            Fires a feed_down webhook to subscribed endpoints, then a feed_recovered webhook when polls succeed again. Add those event types to a webhook above to receive them.

            Embed / dashboard integration

            PiScope can be iframed inside another dashboard via /piscope?embed=1. Cross-origin iframes are blocked by default for safety (clickjacking, etc.). Add the dashboard's origin below to allow it. Format: 'self' (keyword, with quotes) plus URLs like https://home.example.com or http://10.0.0.x:8090 — comma-separated.

            Takes effect on next page load — affects both the main UI and the embedded map view.

            Daily digest

            Once a day at a configurable local time, PiScope rolls up the last 24 hours into a digest (event totals, top types, rare callouts) and delivers it via the channels you enable below. AI commentary is auto-added if the AI tab above is configured.
